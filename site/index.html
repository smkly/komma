product-spec.md
Product Requirements

The new platform focuses on three core principles:

1. Speed over features. Ship the smallest version that solves the problem. Iterate from real usage, not hypothetical requirements.

2. Local-first data. All documents live on disk as plain markdown. Sync is opt-in via git, not required.

3. Human-in-the-loop AI. Every AI suggestion is a proposal. Nothing changes without explicit approval.

Collaboration

Documents go through review, just like code

Submit any document for review on GitHub. Your team comments on the PR. Pull those comments back into Komma, let Claude address them, and push updates -- all from the editor.

  • Submit for Review creates a branch and PR
  • Pull review comments from GitHub into the Edits tab
  • Claude can address review feedback directly
  • Push updates back to the PR in one click
