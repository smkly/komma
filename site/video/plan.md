# Komma Product Demo Video

## Overview
- **Topic**: Komma - AI-powered markdown editor with human approval and git versioning
- **Hook**: What if AI could edit your writing, but only with your permission?
- **Aha moment**: The approval flow -- seeing a diff appear, then accept/reject it
- **Target audience**: Writers, developers, knowledge workers familiar with markdown
- **Length**: ~45 seconds
- **Resolution**: 480p (draft) / 1080p (final)

## Color Palette
- Background: #151722 -- Komma's dark theme
- Primary: #6E57FF -- Komma purple accent
- Secondary: #7ee09b -- green for additions/accept
- Accent: #f08080 -- red for removals/reject
- Muted: #6b6d80 -- dimmed text
- Surface: #1b1d2e -- card/panel backgrounds

## Arc: Build-Up

## Scene 1: Title (~4s)
**Purpose**: Brand introduction
**Layout**: FULL_CENTER
### Visual elements
- Komma logo (purple K glyph)
- Title text "komma"
- Subtitle tagline
### Animation sequence
1. FadeIn logo -- brand moment (~1s)
2. Write title -- name reveal (~1s)
3. FadeIn subtitle -- value prop (~1s)

## Scene 2: Editor (~8s)
**Purpose**: Show the markdown editor with a document
**Layout**: FULL_CENTER
### Visual elements
- Editor window with titlebar dots
- Markdown text content
- Cursor/highlight appearing on text
- Cmd+K badge appearing
### Animation sequence
1. FadeIn editor frame (~1s)
2. Write document text line by line (~2s)
3. Highlight a section of text (~1s)
4. Show Cmd+K popup (~1s)

## Scene 3: Diff/Approval (~12s)
**Purpose**: Core feature -- AI proposes changes, human approves
**Layout**: FULL_CENTER
### Visual elements
- Red strikethrough lines (removed text)
- Green highlighted lines (added text)
- Accept / Reject / Revise buttons
- Check animation on accept
### Animation sequence
1. Transform highlighted text into diff view (~2s)
2. Show removed lines with strikethrough (~1.5s)
3. Show added lines with green highlight (~1.5s)
4. FadeIn approval buttons (~1s)
5. Click Accept -- flash green (~2s)
6. Diff collapses to clean text (~1.5s)

## Scene 4: Version History (~8s)
**Purpose**: Git-based versioning -- every change tracked
**Layout**: FULL_CENTER
### Visual elements
- Timeline with dots and commit labels
- Labels: "Claude edit", "Saved", "PR #14"
- Restore button appearing
### Animation sequence
1. Build timeline from top to bottom (~3s)
2. Highlight entries with badges (~2s)
3. Show restore action (~2s)

## Scene 5: Vault (~8s)
**Purpose**: Cross-document context via @-mentions
**Layout**: FULL_CENTER
### Visual elements
- Chat input with @vault mention
- File tree sidebar
- Claude response referencing other docs
### Animation sequence
1. Show chat input with @mention typing (~2s)
2. Show file references expanding (~2s)
3. Show Claude response (~2s)

## Scene 6: Closing (~5s)
**Purpose**: CTA
**Layout**: FULL_CENTER
### Visual elements
- Logo
- "Free & open source"
- GitHub link
### Animation sequence
1. FadeIn logo + tagline (~1.5s)
2. FadeIn CTA (~1.5s)
